AnyOprSetMuscleRecruitmentNormalization¶

Type

Abstract

Parent class

AnyOprSetMechSolver

Description¶

Class specifying the optimization criterion for muscle recruitment problem, typically solved wihtin an AnyBodyStudy with operations of class AnyMechOprInvMuscleDynAnl.

Basically the criterion for the muscle recruitment use muscle activity as basic variable. We shall here denote this \(a_i\) for the i’th muscle.

\[a_i ~=~ \frac{ f^{(m)}_i }{S_i}\]

where \(f^{(m)}_i\) is the muscle force of the i’th muscle and \(S_i\) it its current strength.

The members of this class allows to modify terms of the criterion by scaling the basic variable of the particular term. Typically, a term have the form of a power function like this

\[Term ~=~ \sum_i \left( a_i \right)^p\]

and the full criterion can be a sum of such terms, please see more details under the class AnyOprSetMuscleRecruitment.

Using the members of this class a term can be modified according to the form:

\[Term ~=~ \sum_i W_i \, \left( k_i \, a_i \right)^p\]

where \(W_i\) is a weight factor for the whole term (regardless of power and the like) and \(k_i\) is a factor multiplied to scale the basic variable used the term \(x_i ~=~ k_i \, a_i\). Notice that also terms that are not part of the objective function of the muscle recruitment criterion, for instance the upper and lower bounds, can be scaled using instances of this class.

A constant common weight factor can be assigned with the member Weight, whereas the member Weight_SearchName can be used to specify a muscle specific weight factor. Weight_SearchName should contain the name of a variable to be found in the muscle (AnyMuscle) objects. \(W_i\) will be the product of both factors.

\(k_i\) is specified by Type member as well as the Nfactor_SearchName. As with the term weight factor, the resulting \(k_i\) is the product of factors arising from the specification of Type and Nfactor_SearchName. Type and Nfactor_SearchName specifies the force normalization \(N_i\) such that

\[x_i ~=~ \frac{ f^{(m)}_i }{N_i}\]

In other words,

\[k_i ~=~ \frac{ 1 }{N_i}\]

The Type member specifies a number of common (built-in) normalization methods, see the table below, whereas Nfactor_SearchName gives the option to specifying a muscle specific factor; like Weight_SearchName it should contain the name of a variable to be found in the muscle (AnyMuscle) objects.
