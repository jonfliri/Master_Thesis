AnyKinLinear¶

Type

Creatable

Parent class

AnyKinMeasureReal

Child classes

AnyKinCoM

Description¶

This “kinematic measure” measures translation, i.e., in principle a 3-D vector. It is either the position of one reference frame with respect to the global frame or the relative translation between two frames. By default the measure output consists of the three global components of the vector, but by the Ref member you may specify another frame of reference frame and with the Type member you can select other options.

Cartesian coordinates (Type = CartesianCoord):

Cartesian coordinates, Pos=[\(x\), \(y\), \(z\)] with optional choice of frame of reference, is the default output.

Cylindrical coordinates (Type = CylindricalCoord):

This option provides a transformation of the Cartesian coordinates into cylindrical coordinates, Pos=[\(r\), \(\theta\), \(e\)]. The AngularAxis member specifies along which axis the elevation, \(e\) is measured and thus the normal to the plane in which \(r\) and \(\theta\) is measured The axis from which the angle, \(\theta\), is measured also depend on AngularAxis as follows:

Type = x:

\(\theta\) is measured from the y-axis

Type = y:

\(\theta\) is measured from the z-axis

Type = z:

\(\theta\) is measured from the x-axis

Currently, the CylindricalCoord option does not obey the Ref member.

Spherical coordinates (Type = SphericalCoord):

This option provides a transformation of the Cartesian coordinates into spherical coordinates Pos=[\(r\), \(\theta\), \(\phi\)]. The angle \(\theta\) is the same as for cylindrical coordinates, see above. The AngularAxis member specifies about which axis \(\theta\) is measured and the zero axis. \(\phi\) is the angle between the Cartesian vector and the plane given by AngularAxis; \(\phi\) is positive in the half-plane of the AngularAxis.

Currently, the SphericalCoord option does not obey the Ref member.

Line-to-line distances (Type = LineLineDistance):

This option measures the distance between one axes of the each of the two input frames. The member LineLineDistanceAxes specifies which pairs of axes enter the measurement; nine different combinations are possible, but the default is x-x.

The pure line-to-line distance measure has a built-in singularity issue. For two infinite long lines in the same plane the distance is zero, unless the lines are perfectly parallel. This implies that there is an inherent discontinuity the definition. In order to smoothen this, the lines are not infinitely long in implementation of AnyKinLinear. The LineLineDistanceAxisLength specifies the finite length of the second axis.

In spite of this smoothening, it is recommended to use the option with care when the axes are close to parallel and in plane. For axes that are supposed to be parallel, Cartesian coordinates may be a better option to obtain the equivalent measurement

This measure is used in the standard joints as well as for describing external loads, such as point loads. The measure can also be used for driving a motion of a point, by adding a kinematic driver to an AnyKinLinear object.

Equivalent forces¶

Forces can be applied to AnyKinLinear objects by AnyForce objects, and they come implicitly from many other objects such kinematic constraints. Here, we shall briefly discuss the meaning for the associated forces, but only for the case of Cartesian coordinates, because they are the most frequently used. Forces related to the other coordinate output are more complicated to interpret and beyond the scope of this text.

The principle of virtual work tells us intuitively that forces associated with an AnyKinLinear object using Cartesian coordinates must be a linear force, i.e., a 3-d vector, a point load. The kinematic measure is a vector from frame origin ‘0’ to ‘1’. This implies that the components of the generalized force (F in AnyForce) is equal to the force applied on frame ‘1’ from frame ‘0’. However, there are more loads associated with the generalized force.

A mechanical explanation of these loads can be given by considering the AnyForce applied to the AnyKinLinear as rigid bar from frame ‘0’ to frame ‘1’. In order to control the force on frame ‘1’, the rotation of the bar must be controlled by some (virtual) rotational actuators; indeed these will apply moment reactions on some part in the system. From the principle of virtual work it follows that this part is the reference frame that is the basis of the measure. Notice that this frame by default is the global frame; in other words a torque could be transferred to the global frame in this case, or one of frame ‘0’ or ‘1’ if local components of one of these frames are used to measure the position vector. This issue is known to cause some confusion about the force applied to an AnyKinLinear object; however, this mechanism also makes sure that over all equilibrium is ensured.
