AnyProjectModelViewDefinition¶

Type

Creatable

Parent class

AnyProjectDescriptionObject

Description¶

Class for AnyProject related Model View Definitions.

Model View Definitions are objects that define a view of the model. The view is defined by associating a number of draw groups with a set of Model View state actions. During activation of the view definition, the state actions will be performed on the drawing groups in the sequence that the drawing groups are provided to the Model View Definition object. Please recall that a drawing group is made by class AnyDrawGroup.

The DrawGroupSequence member must be filled with the drawing groups on which the actions should be performed. The actions are defined by a number of arrays (Reset, Hide, ShowModelDefined, ShowAutoGenerated, Transparent, Select) that holds switches for whether to perform the particular action to the associated drawing group (On) or not (Off). These arrays must have the same or smaller number of elements as the DrawGroupSequence member, with each element of these arrays affecting the draw group with the same index in the DrawGroupSequence member. If an array length does not match the DrawGroupSequence length, the non-specified action array indices are regarded as having the Off-value. It is legal to not define values for an action switch array, in which case the array values will be Off for all draw groups. A Model View Definition can be constructed to apply the same actions to all draw groups in the DrawGroupSequence member by only defining a maximum of a single On/Off value for all switch arrays. In that case the action switch arrays will use this single value for all draw groups in DrawGroupSequence.

The Model View Definition is applied to the model view in the following way: First the contents of the model view are all hidden and deselected. For each draw group in the DrawGroupSequence member, the following state actions are performed in fixed order, but only if the state action is active for this draw group: {Reset, Hide, Show ModelDefined, Show AutoGenerated, Make transparent, Select}.

Model View Definitions are typically defined in the Views folder of an AnyProject object.
