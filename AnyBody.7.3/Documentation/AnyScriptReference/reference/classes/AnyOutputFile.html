AnyOutputFile¶

Type

Creatable

Parent class

AnyOutputFileBase

Child classes

AnyOutputBVH, AnyOutputGCD

Description¶

This class provides functionality for saving output data to a file from a study (AnyStudy or derived) during execution of one of its operations. The members of this class are used to define the file name, its format, and the variables that should be reported in the file.

Notice that the AnyOutputFile object must be declared inside the study in order for the study to create the file. If you wish to use the same output file specification in multiple studies, you can do this by references from studies to output file declarations made outside the studies. Such references will make the studies treat the output file objects as if they were declared inside the studies.

You can specify the output data in two ways. Basically, all variables in an output-file object will be included into the list of output and thereby enter the data file. Thus, you can declare new value-objects inside the output-file-object and they will enter the file. You can also enter variables by pointers. This is done by “Values” member that is a pointer array, which for instance can be filled using AnyScript search functions.

Alternatively, values can be found by a variable-search available in the object. The “Search” member is an AnyString that can hold one or more of such complete name variable-search strings. This search option is similarly to the one available the Chart View of the AnyBody GUI. Wildcard characters (*,``?``) can be used to perform advanced selections.

The output-file object distinguishes between values that change during the operation and constants. Constants will be reported only once for instance in the header section of the file, whereas changing values will be reported for each step of the operation. The study that owns the executed operation automatically finds which values are changing and which are constant (this is similarly to the automatic identification of values in the study’s ‘Output’ folder).

The following formats are currently supported; please notice that the extension of the ‘FileName’ member does not affect the format of actual file:

Text (CSV)

Column formatted text output. Variables are reported in columns with one row for each operation step. The file header contains constants and column descriptions. Column separation sign is specified with the ‘SepSign’.

XML

XML formatted text output. Variables are reported in a table (xhtml formatted) with a table-header containing the variable names. The file header contains constants in tags named AnyValue; the constant’s name is given by an attribute and the value is saved between start and end tag.

Future versions

It is considered to include other output formats in a future version. The following formats are being considered:

filename.dat

Binary format.

filename.any:

AnyScript formatted file, which can directly be used as an include file.

filename.m:

Matlab M-file format that makes the data directly accessible from Matlab.

filename.py:

Python script file format.

Suggestions are very welcome from users who miss functionality from the AnyOutputFile.
