AnySurfFile¶

Type

Creatable

Parent class

AnyFile

Child classes

AnySurfFileArray, AnySurfFileVar

Description¶

Class for file objects that hold geometrical surfaces.

The format of the surface files is basically STL (ASCII). However, AnyBody has its own format with the extension “.anysurf”. AnySurfFile objects creates the ANYSURF file and perform the conversion from STL to ANYSURF automatically, if needed. The need for updating the ANYSURF file is based on the modification time of the files; if the ANYSURF file does not exist or is older than the STL file, it is updated.

Please notice that the conversion may take some time. The benefits of the ANYSURF files are:

  1. They are smaller than STL

  2. Special information required by AnyBody does not need to be extracted each time you load a model. This is information such as vertex normal, which is utilized for rendering and contact evaluation. This saves time, except when the STL files have been changed.

New in version 5.0: AnyBody, v.5.0 introduced a new version and format of the ANYSURF file. This is in principle a binary format aimed at increasing loading speed. In order to provide backwards compatibility the binary data is put in a section after the ASCII data known from the older version of ANYSURF files. Thereby, the new format can still be read by older AnyBody versions, whereas newer version can exploit the binary section, skipping the ASCII section.
