AnyOptStudy¶

Type

Creatable

Parent class

AnyDesStudy

Child classes

AnyOptKinStudy

Description¶

This study does optimization. It varies design parameters systematically with the aim of optimizing a certain criterion. Additionally, constraints can be applied to the process.

Mathematically, the optimization can be formulated as

\[\begin{split}\begin{equation} \begin{array}[c]{lcl} minimize & F(\VEC{x}) & ~ \\ ~~~~\VEC{x} &&\\ &&\\ subject~to & g_j(\VEC{x}) \leq 0, & j=1,m \\ & h_k(\VEC{x}) = 0, & k=1,l \\ & x_i^{(l)} \leq x_i \leq x_i^{(u)}, & i=1,n \\ \end{array} \end{equation}\end{split}\]

where \(\VEC{x}\) is a vector of the \(n\) design variable defined by AnyDesVar objects. \(F\) is the so-called objective function, which is defined by correct type specification of one of the AnyDesMeasure objects. The constraints, \(g_j\) and \(h_k\), are also defined proper type specification of the remaining AnyDesMeasure objects. The final row is the simple bounds on the design variable \(x_i\); the lower and upper bounds, \(x_i^{(l)}\) and \(x_i^{(u)}\), are defined for each design variable in the AnyDesVar objects.

Please notice that none of the current solvers handle equality constraints, so this option in the AnyDesMeasure is currently only a preparation of the system for future use.

The optimization processes inside AnyOptStudy is handled as a sequential process doing repetitive step, each improving (or trying to improve) the design. The basic independent parameter (abscissa value) in the optimization operation is the iteration step and the output of design variable values and design measures are stored for each such design step in the Output folder.

The optimization solver is configured in the operation member, Optimization. Current AnyOptStudy provides the following solvers.

Simple search:

(Optimization.Solver = OptSimpleSearch) This method employs a simple scheme searching in the direction of each of the design variables in turn. A golden-section line search is used in each direction.

Steepest descent based feasible direction search method:

(Optimization.Solver = OptFeasibleDirSearch1) This method is based on the steepest descent search method approach for unconstrained optimization problems. In order to account for constraints, the search direction is modified to avoid constraint violations. A golden-section line search is used in the obtained search direction to improve the design point.

The method is uses numerical sensitivity analysis for finding gradient; the perturbation is configurable in the solver settings. Furthermore, settings for convergence control and constraint activation handling is available in the settings.

Conjugate direction based feasible direction search method:

(Optimization.Solver = OptFeasibleDirSearch2) This method is the same as the previously mentioned, but with the small modification that it employ the well-known conjugate direction modifications of the search direction to increase the speed of convergence in ill-conditioned problems. This modification is only applied to regions where steepest descent method is previously used, i.e., regions with no active constraints.

Complex method:

(Optimization.Solver = OptComplexPenalty) The Complex method is a simple and robust 0th-order algorithm that searches for the optimum by systematically shrinking a population of design points until they meet at the optimum. This approach is very different compared to the gradient-based search algorithms and it is often lesser sensitive to non-smooth problems than gradient-based methods. Therefore, it can be a powerful alternative to the above-mentioned optimizers.

For the Complex solver, an initial population is randomly generated around the initial design point. The initial population is generated within a box around the initial design point. The population and box sizes can be given in the solver settings. Notice that the population size must be at least one more than the number of design variables.

The method operates by moving the worst design point according to some scheme. The alpha parameter spoecifies a dimensionless step-length and futhermore a parameter specifies the maximal number of attempts moving a given point before a new random point is selected to substitute it. Both these parameters a also in the solver setting, but their default values are recommended values that typically needs no adjustment.

The Complex method is inherently an unconstrained method, but constraints can be enforced via penalization. Constraint penalization can be added manually (i.e. by the user) to the objective function value or a polynomial penalization function built-in the Complex solver can be used. The built-in polynomial penalization is governed by a factor and the exponent, which can be set in the solver settings.

Future versions

We acknowledge that the currently available facilities in the AnyOptStudy are limited and we must emphazise that they may not solve any possible optimization problem. Optimization is a complicated field and indeed mechanical and biomechanical problems can be particularly difficult to deal with. The AnyOptStudy has been made available in AnyBody version 2.0 for the first time and depending on the experiences, we will aim at developing it further in future releases.

An alternative option to the facilities in AnyOptStudy is to use AnyBody’s console application together with an external optimization routine.

We are always prepared to consider extentions of these optimization facilities arising from actual demands and will be grateful for any requests and comments from the users.
