AnyMechOprKinAnl¶

Type

Creatable

Parent class

AnyMechOpr

Description¶

This operation-class performs kinematical analysis of a mechanical systems. The mechanical system is defined a mechanical study object (class AnyMechStudyBase or derived) and the kinematical analysis object must be member of a mechanical study.

The default kinematical analysis requires the system to be kinematically determinate, implying that there must be enough kinemtical constraints to determine all degrees of freedom of the system.

For such cases, the analysis assembles and solves the kinematic constraints for each requested time step. The position analysis solves the non-linear constraint equations by a Newton-Raphson like method. This means that it, to some extend, relies on the initial configuration of the system. In the first step of an analysis, this is the loading time positions of the segments, whereas in all subsequent time steps, the initially guess is based on the previous known values of positions, velocities, and accelerations of the system.

The equation solver handles redundant kinematic constraints, which means that the system can appear to be over-determinate, but only if the excess of equations are dependent of the other equations the system can be accepted.

The kinematical analysis can be executed in two modes controlled by the switch SingleStepOnOff: A single step mode, where the analysis is performed for a single time step of the study (by default the first step), and a multi-step mode, where it sequentially solves the kinematical equations for each time step defined in the study.

By default, velocity and acceleration analysis are also carried out; however, they can be bypassed if only position information is needed from the analysis (see member ‘PosAnalysisOnlyOnOff’).

The kinematical analysis has a number of solver options. From version 4.1, these options have been selectable by the member ‘SolverType’, of which one handles over-determinate kinematical systems, i.e., a mechanical system with more constraints than rigid-body degrees of freedom of the segments.

The over-determinate kinematical solver minimizes the norm of the so-called soft constraints subject to the so-called hard constraints. Constraints are specified as either “soft” or “hard” in the AnyKinEq-object that defines the constraints. The over-determinate solver basically only performs position analysis in the current implementation. It is, however, extended with a numerical approximation of velocities and accelerations, which is partially done using finite difference differentiation.
