AnyKinStudy¶

Type

Creatable

Parent class

AnyMechStudyBase

Child classes

AnyBodyCalibrationStudy, AnyBodyStudy, AnyMechStudy

Description¶

This study class comprises operations for kinematical analysis.

The kinematical analysis requires the system to be kinematically determinate, implying that there must be enough kinematical constraints to determine all degrees of freedom in the system.

The kinematical analysis is carried out by operations of the class AnyMechOprKinAnl. The InitialConditions operation is a single step kinematical analysis that is aimed at assembling the mechanism for initial positioning.

The Kinematics operation performs sequentiel analysis of all time steps defined in the study. The Kinematics operation uses InitialConditions as pre-process, so the mechanism is assembled robustly before starting the analysis.

By default, the InitialConditions is using the same solver as the Kinematics. This behaviour can be changed by setting the InitialConditions.Solver variable explicitely.

Please study the AnyMechOprKinAnl class for more details on the kinematic analysis.

Replay¶ Operation that replay previously calculated results stored in the Output...

Class type: AnyOperationReplay
Eval moment: Const
Default: {}

Operation that replay previously calculated results stored in the Output folder. It also replays data in the Output folder if they are imported from an output storage file. Replaying means to transfer data from the Output folder back into their original value objects in the model. This restores all saved value as they were upon the time of calculation. The Replay operation executes the same number of time steps as the original simulation.
