AnyMuscleModel¶

Type

Creatable

Parent class

AnyConstitutiveModel

Child classes

AnyMuscleModel2ELin, AnyMuscleModel3E, AnyMuscleModelUsr1

Description¶

Simple muscle model class.

This is the base class for all muscle models, i.e., the constitutive model of a muscle. A muscle-model object provides information about the muscle strength, passive forces, and the assumed relationship between passive and active forces to the muscle object.

AnyMuscleModel is not an abstract base class. It contains the simplest possible muscle-model that features a constant strength without any passive element properties.

Note

The input parameters F0, Vol0 and Lf0 are independent inputs. It is possible (and recommended) to link these with suitable expressions depending on the information that is avaiable for your model.

The variable F0 could be implemented as function of a nominal muscle stress capability times PCSA (physiological cross sectional area), which again could be calculated from Vol0 and ‘Lf0’ (e.g. \(F0 = constant \times Vol0 / Lf0\)). Such explicit implementation of the relationship between the input, enable the muscle strength to be updated by the system as part of for instance model scaling and calibration steps.

More details on muscle models and calibration can be found the tutorial about muscle modeling.

Examples¶

Here is simple example of a muscle with the basic parameters defined.

AnyMuscleModel MyMuscle = {
    Lf0 = 0.1; // Optimal fiberlength in meters
    Vol0 = 0.4; // Muscle volume in cubic meters
    F0 = 100; // Maximum force output in newtons
};

Note

The muscle volume is not used directly in this example since F0 does not depend on it. However, some types of muscle recruitment may still use the Vol0 parameter as a normalization factor.
