AnyForceMomentMeasure¶

Type

Creatable

Parent class

AnyForceMeasure

Child classes

AnyForceMomentMeasure2

Description¶

This particular class “measures” forces outputs an equivalent point-force and a moment, i.e., two geometrical vector quantities in global coordinates.

All forces specified (or referenced) inside an instance of this class will be summed up automatically to a single force and moment vector. A point-of-interest is given by a reference frame (i.e. AnyRefFrame or derived).

Only forces that are applied directly to this reference frame will be included. Therefore, this measure is used to investigate the effect of forces applied to a specific point; also if you are simply in doubt about a force’s point of application.

The point-of-interest and the forces to measure are specified by two member object-pointers, ‘RefPoint’ and ‘IncludeForces’. Notice, that earlier versions of AnyBody found these forces and the point-of-interest from the members in the object (as so-called Expected Members). This old-style functionality is still supported, but mistakes can easily be made with these Expected Members input, so it is strongly recommended to specify the object-pointer directly. If ‘RefPoint’ and ‘IncludeForces’ are specified, the “Expected Members” are ignored.

The force-measurement functionality is based on the principle of virtual work, using virtual displacements of the reference frame, i.e. the point-of-interest, and any nested frames. The rest of the mechanical system is not moved by this displacement. This displacement may violate kinematic constraints in joints, etc., and therefore, forces transmitted by the joints are neglected, unless the joint is attached to the point-of-interest. Consequently, this measure does not measure the effects of any given force at a point-of-interest “far away” from the application point, i.e. if they are not directly connected.
