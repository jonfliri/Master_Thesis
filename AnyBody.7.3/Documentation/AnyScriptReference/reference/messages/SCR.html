SCR.SCN5¶

Severity

Fatal error

Error

‘Additional error description’

Documentation Comments are a special kind of AnyScript source code comments. Apart from being a comment, in the sense that the text is not compiled by the AnyScript parser, a Documentation Comment is intended to contain the script writer’s “documentation” of a specific object in the model. It is, therefore, treated differently than normal source code comments during loading the model.

The Documentation Comment will be associated and stored with an object and it is typically be written in the code either right before or after the particular object. The Documentation Comments can be made in front of an object (prefixed) by

  1. /// ... instead of // ...

  2. /** ... */ instead of /* ... */

And after an object (postfixed) by

  1. ///< ... instead of // ...

  2. /**< ... */ instead of /* ... */

Combinations of 1 and 2, respectively 3 and 4, are allowed if no linespace is dividing the individual blocks. Space in the form of empty lines between the object and the comment are not accepted.

A third option is the so-called scope-fixed Documentation Comment that is attached to the object of the scope the comment is placed. This is useful for placing comments on an object from include files or from conditional sections of the code.

Scope-fixed Documentation Comment are made by

  1. ///^ ... instead of // ...

  2. /**^ ... */ instead of /* ... */

Multiple Documentation Comments, for instance before and after an object, comments attached to multiple initializations of folder, and one or more scope-fixed comments, are allowed. In all such cases, the comments are internally assembled to one Documentation Comment.

Unexpected character.

This error indicates an AnyScript syntax error. It is indicating that AnyScript scanner does not have rules for accepting the character in that place of the code and no other, more particular, error is detected.

SCR.SCN7¶

Severity

Fatal error

Error

Identifier already defined at : ‘FileName(LineNo)’

The AnyScript scanner is indicating that an already defined identifier is being defined again. The identifier cannot be redefined. The location of the previous definition is attached to the message.

SCR.SCN8¶

Severity

Important warning

Error

#define- or #path-statement conflicts with predefined global identifier name. Conflicting object : ‘ObjectName’. The global variable will be inaccessible in the script, because the preprocessor definition have higher priority.

The AnyScript scanner has identified that an illegal attempt to redefine a global identifier occured. This occurs when global identifier names are used in #define and #path statements Such redefinitions make the global variable inaccessible because preprocessor statement has higher priority by the AnyScript parser. It is not recommended to allow such conflict, but they may not cause serious problems.The variable may still serve its purpose in the model, but you will not be able use it explicitly in the script.

Severity

Path does not exist.

SCR.SCN10¶

Severity

Fatal error

Error

Path definition does not exist.

The AnyScript scanner is indicating that reference to a non-exisitng path definition was found. For example the following #include-statement

#include “<PATHDEFINITION>PathPart2.any”

without a previous

#path PATHDEFINITION …

will lead to this error.

SCR.SCN12¶

Severity

Fatal error

Error

Path or globally predefined identifier name cannot be undefined by #undef.

The AnyScript scanner is indicating an attempt to undefine a global identifier. Globally defined identifier cannot be undefined or redefined. Also #path-definitions cannot be undefined and attempt to do so will lead to this error.

SCR.SCN17¶

The AnyScript scanner is indicating that not all template arguments are provided when using the Class Template. Template arguments, that has no default values given in Class Template definition, must be initialized at the Class Template usage point.

SCR.SCN22¶

Severity

Fatal error

Error

Defined #path can not be used as define or class template argument identifier. See definition : ‘TokenString’ : ‘FileName(LineNo)’

This error indicates an AnyScript syntax error. Defined #path holds its value as global preprocessor definition.Therefore it can not be used as define or class template argument identifier which would override it locally.

SCR.PRS5¶

Severity

Fatal error

Error

Objects of class ‘ClassName’ cannot be created because it is an abstract class

Abstract classes do not contain full functionality and therefore objects hereof the cannot be created. Abstract classes merely serve to provide a common base for derived classes. Abstract classes can be used as references. Try to find the class that really provide the functionality, you are looking for and use this instead. Or maybe you intended this to be a reference to an ‘ClassName’ object.

SCR.PRS6¶

Severity

Fatal error

Error

Class name expected

This error arise when making a declaration of an objects using a wrong class name. Typically, this is due to spelling mistake. Notice that the syntax highligting will indicate when the class name is know to the parser. Only creatable and abstract classes are known to the parser, but there do exist some certain “hidden” classes that are not know and these cannot be used anywhere in AnyScript as class names.

SCR.PRS9¶

Severity

Fatal error

Error

Unresolved object

An unresolved object is an object name for which a matching object in the AnyScript data structure was not found. This could for instance be due to a typing mistake in the object name or a reference to an object that was deleted from the model. Notice that mistake could be in any part of a long name, i.e., a name pointing through several scopes, even though only the last part of the name is included in the message.

SCR.PRS10¶

Severity

Fatal error

Error

Initialization denied

This error implies that the model script contains a explicit initialization of an AnyValue with restricted access. Initialization of the particular value is not alowed. Typically, it is a computational value that the model uses to store result and as such it value is controlled completely by the system.

SCR.PRS11¶

Severity

Fatal error

Error

‘TokenString’ unexpected

This is general error that occurs when the basic syntactical principles of AnyScript are not followed and no other, more particular, error is detected.

Unexpected ‘end of file’ (EOF) is a special case, in which the end of a file is reached before expected. This happens when start and end scope delimiters do not match pairwise, so that the end of the file is reached, while still in a scope opened in the particular file.

Circular reference

SCR.PRS23¶

Severity

Fatal error

Error

Expression assignments are not allowed in declarations of AnyFolder objects

Folder objects cannot be assigned with an expression. In this particular case, the error occur during in the declaration of the object. Maybe a reference operator is missing if you wanted to make a reference to another objects. Possible syntax:

<folder_class_name> &<reference_name> = <object_name (complete or relative)>;

<folder_class_name> <object_name> = { … };

where the former is a reference declaration and assignment and the latter is a folder declaration and assignment.

SCR.PRS24¶

Severity

Fatal error

Error

Folder assignments are not allowed in declarations of AnyValue object

Value objects cannot be assigned with folder scope, {}. In this particular case, the error occur during in the declaration of the value object. Maybe you class name specifier is wrong. Possible syntax:

<value_class_name> <object_name> = <expression>;

<folder_class_name> <object_name> = { … };

where the former is a value declaration and assignment and the latter is a folder declaration and assignment.

SCR.EXP10¶

Severity

Fatal error

Error

Expression evaluation failed at moment ‘EvaluationMoment’ : ‘FileName(LineNo)’ : ‘ObjectName’ : argument will not be ready for evaluation until moment ‘EvaluationMoment’

All values in AnyBody are assigned to specific Evaluation Moments that specify when the values are evaluated. All the Evaluation Moments together define a kind of time line of execution of the model from the early model loading steps to steps in running operations (AnyOperation). This error indicates that an AnyScript value has been assigned with an expression containing other symbolic variables of a later Evaluation Moment. This is not allowed because the dependent value, in this case, would be updated before the inputs, i.e., with “old” values.

SCR.EXP12¶

Severity

Error

Error

Error in expression for preprocessor variable. Evaluation skipped.

This error arise from bad expressions used for preprocessor variable assignments. “Evaluation skipped” implies that the expression is not evaluated and therefore the default value for the particular variable is retained. The preprocessor variables are evaluated early in the loading process (during parsing) and only simple error checking is available at this stage. This error is typically followed by other related errors, either from more detailed expression error handling later or errors that follows from the faulty expression producing bad values. Therefore, all following errors should be investigated and if possible resolved first.

SCR.EXP.FUN.EXT1¶

Severity

Fatal error

Error

AnyFunEx is missing ‘Return’ member.

AnyFunEx must contain AnyValue derived class object ‘Return’. Class and dimensionality of this object gives information about what type AnyFunction returns when used.

SCR.EXP.FUN.EXT2¶

Severity

Fatal error

Error

AnyFunEx return type : ‘ClassName’ : is not supported.

The type of the return value is not supported by the AnyFunEx interface. Only types with simple conversions to the external code are currently supported. These types are AnyInt, AnyFloat, AnyString.

SCR.EXP.FUN.EXT3¶

Severity

Fatal error

Error

AnyFunEx argument type, ‘ClassName’, is not supported.

The type of an argument value is not supported by the AnyFunEx interface. Only types with simple conversions to the external code are currently supported. These types are AnyInt, AnyFloat, AnyString.

SCR.EXP.FUN.EXT4¶

Severity

Fatal error

Error

AnyBody.Python extension module is not installed or does not match installed version of Python.

This error arises if neither of AnyBody’s Python interfaces matches any of the installed Python versions. AnyBody’s binaries contain the folder called AnyBody.Python, which contains the Python interfaces, such as AnyBody.Python.2.x.dll for version 2.x of Python. If a matching dll for the installed version of Python does not exist, interfacing cannot happen.
