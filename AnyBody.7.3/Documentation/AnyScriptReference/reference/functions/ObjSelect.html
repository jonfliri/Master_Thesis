ObjSelect()¶

Implemented by

AnyFunObjSelect

Description¶

Select function for finding objects in the model.

The function returns an array of object pointers (class AnyObjectPtr) matching the search criterion. The search criterion is given by a variable number of selection strings.

The selection strings can contain the following.

  • Search strings that are executing searches in the object tree by name with wildcard characters (,?) allowed. The ‘’ character indicates “any substring” in the given name token, whereas the ‘?’ character indicates “any single character”.

  • Value condition strings that are applied as test to the objects in current result. If the test evaluates as false the object is removed from the results. Value condition strings starts with one of the following operators: ‘=’, ‘<’, ‘>’, ‘<=’, ‘>=’, ‘!=’. C++-style ‘==’ operator is also handled and treated as equality test similarly to a single ‘=’, and ‘<>’ is like ‘!=’ treated non-equal. Some example test strings are:

    • Single floating point value test: AnyString test = “=10.0”;</li>

    • Single string value test: AnyString test = “=My Name”;</li>

    • Test on a vector of integers: AnyString test = {“=1”,”=2”,”=3”};</li>

    • Empty test, which is always true: AnyString test = “”;</li>

    Notice that tests on multiple element data structures is conducted on each element separately. The test on all data elements must be true for a combined test to be true. A data element with no associated test string is regarded as a true test whereas as test string without data element is treated according to the operator. For this special case ‘not equal’ returns true while all other test return false.

The first selection string is applied in the scope where the function is called, i.e., at the place in the Model Tree of the current object. The following selection strings, whether it being search or condition operations, are applied sequentially to the result of the previous operation.

This function only searches in the current folder.
